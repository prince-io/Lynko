# models — Mongoose Schemas

## Ownership

Owns all three Mongoose models: `User`, `Lynko`, `Design`.

## Local Contracts

- All models use the `mongoose.models.<Name> || mongoose.model("Name", schema)` singleton pattern
- `User` and `Design` identify users by `clerkUserId` (String, unique, indexed)
- `Lynko` identifies users by `userId` (String, indexed, not unique — same Clerk ID value, different field name)
- `Design.customization` is `Mixed` type with a `default` object — keep the shape in sync with `api/designs/route.js` defaultDesign

### Schema details

**User**
- `clerkUserId` (required, unique, indexed)
- `username` (required, unique, indexed) — autogenerated on first dashboard visit
- `bio` (String, default "")
- `profilePic` (String, default "") — Cloudinary secure_url
- timestamps: true

**Lynko**
- `userId` (required, indexed) — Clerk user ID
- `title` (required, trim)
- `url` (required)
- `order` (Number, default 0)
- `isActive` (Boolean, default true)
- timestamps: true

**Design**
- `clerkUserId` (required, unique, indexed)
- `customization` (Mixed) — default object with theme, font, size, radius, border, avatar, background, buttonStyle, buttonRadius
- timestamps: true

## Work Guidance

- Never rename `Lynko.userId` to `clerkUserId` — the Lynko model intentionally uses a different field name
- When adding fields to `Design.customization`, update both the model default and the API route default simultaneously
- Default value shapes are duplicated — check both locations when editing defaults
